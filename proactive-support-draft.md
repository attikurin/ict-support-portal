## 1. 自発的・能動的支援の定義

### 1-1. 基本的な定義

**自発的な支援（Voluntary Support）**
- **自分から進んで**支援を提供すること
- 依頼される前に行動を起こす
- 問題を予測し、事前に対応する

**能動的な支援（Proactive Support）**
- **積極的に**働きかける支援
- 待つのではなく、自ら動く
- 継続的な改善を提案・実行する

### 1-2. ICT支援員における具体的な意味

#### 受動的支援（Reactive Support）
```
【従来型の支援】
先生：「プロジェクターが映らないんですが…」
支援員：「はい、今すぐ見に行きます」
→ 問題が発生してから対応
```

#### 能動的支援（Proactive Support）
```
【能動的な支援】
支援員：「来週の授業でプロジェクターを使うと伺いました。
        事前に動作確認をしておきましょうか？」
先生：「ありがとうございます！助かります」
→ 問題が発生する前に予防
```

## 2. 受動的支援との違い

### 2-1. 比較表

| 観点 | 受動的支援 | 能動的支援 |
|------|-----------|-----------|
| **姿勢** | 待つ | 動く |
| | 依頼されたら対応 | 自ら提案・実行 |
| | 指示待ち | 先読み行動 |
| **タイミング** | 問題発生後 | 問題発生前 |
| | トラブル対応 | トラブル予防 |
| | 緊急対応 | 計画的対応 |
| **範囲** | 依頼された内容のみ | 関連事項も含めて |
| | 個別対応 | 全体最適 |
| | 短期的 | 長期的 |
| **関係性** | 「困ったら呼ぶ人」 | 「一緒に考える人」 |
| | サービス提供者 | パートナー |
| | 距離感あり | 信頼関係 |
| **成果** | その場しのぎ | 根本解決 |
| | 同じ問題が再発 | 再発防止 |
| | 個人の成長止まる | 組織全体の成長 |

### 2-2. 具体的なシーン比較

#### シーン1: 授業支援の場合

**受動的支援**
```
授業開始5分前
先生：「今日Google Classroomを使いたいんですが、やり方が…」
支援員：「今から説明しますね」
→ 授業開始が遅れる、先生も焦る、子どもたちも待つ
```

**能動的支援**
```
前日または数日前
支援員：「来週の授業予定を拝見したのですが、
        Google Classroomを使う予定はありますか？
        事前に使い方をご説明しましょうか？」
先生：「実は使ってみたいと思っていたんです！」
→ 余裕を持って準備、先生も自信を持って授業に臨める
```

#### シーン2: トラブル対応の場合

**受動的支援**
```
授業中
先生：「端末が起動しない子が5人もいます！」
支援員：「今すぐ見に行きます」
→ 授業が中断、他の子どもたちも待つ、授業計画が狂う
```

**能動的支援**
```
授業前日
支援員：「明日の授業で端末を使うとのことで、
        全端末の充電と動作確認をしておきました。
        2台バッテリーが劣化していたので交換しておきます」
先生：「いつもありがとうございます！」
→ トラブル未然防止、授業がスムーズ
```

#### シーン3: 情報提供の場合

**受動的支援**
```
先生：「授業でプログラミングをしたいんですが、
      何か良い教材ありますか？」
支援員：「Scratchが良いと思います」
→ その場限りの情報提供
```

**能動的支援**
```
支援員：「来月から5年生でプログラミング単元が始まりますね。
        他校の実践事例を3つまとめてみました。
        Scratch、Viscuit、プログル、それぞれの特徴と
        単元に合った使い方を資料にしたのでご覧ください。
        必要なら模擬授業もしましょうか？」
先生：「ここまで準備してくれたんですか！助かります！」
→ 包括的・先回り・継続的支援
```

## 3. 自発的・能動的支援の具体例

### 3-1. レベル別具体例

#### 【初級】気づきと声かけ
- 廊下で会ったら「何かお困りのことはありませんか？」
- 職員室で「今週の授業予定、拝見しました」
- 授業前に「準備、お手伝いできることはありますか？」

#### 【中級】予測と準備
- 年間行事予定を確認し、ICT機器が必要な行事を事前リストアップ
- 学期末に次学期の授業で使えるツール情報を提供
- 各教室の機器配置図を作成・更新

#### 【上級】提案と改善
- 校内ICT活用状況を分析し、課題と改善案を提示
- 他校の優良事例を収集・紹介
- 先生方向けの簡単なマニュアルや動画を自主制作

### 3-2. 業務別具体例

#### A. 日常業務での能動的支援

**朝の巡回**
```
【受動的】職員室で待機
【能動的】
・各教室を巡回し、機器の状態を確認
・充電忘れの端末を発見→充電開始
・黒板に書かれた授業予定をチェック
・Wi-Fi接続状況を確認
```

**授業準備**
```
【受動的】呼ばれたら準備を手伝う
【能動的】
・週案や時間割を確認し、ICT活用予定授業をリストアップ
・必要な機器を事前に教室に配置
・先生に「準備完了しました」と報告
・初めて使う機能なら事前レクチャーを提案
```

**授業支援**
```
【受動的】授業中に呼ばれたら駆けつける
【能動的】
・授業開始前に教室へ訪問
・「何かあればすぐ来ますので、遠慮なくお声がけください」
・授業中は近くで待機（邪魔にならない位置）
・トラブルの予兆を察知→事前に対処
```

**授業後フォロー**
```
【受動的】授業が終わったら機器を片付ける
【能動的】
・授業の振り返りを一緒に実施
・「今日の授業、こんな工夫ができるかもしれません」
・次回の授業に向けた改善提案
・良かった点を具体的にフィードバック
```

#### B. 計画的な能動的支援

**月次レベル**
```
・翌月の行事予定を確認
・必要な機器・準備のリストを作成
・校内研修の企画・提案
・ICT活用状況の簡単な集計と共有
```

**学期レベル**
```
・次学期の単元計画を確認
・新しい教材・ツールの情報収集
・長期休暇中の機器メンテナンス計画
・研修資料の作成・更新
```

**年間レベル**
```
・年間のICT活用計画を支援
・機器更新計画への提言
・校内ICT環境の改善提案
・成果報告書の作成支援
```

#### C. 情報提供での能動的支援

**定期的な情報発信**
```
【受動的】質問されたら答える
【能動的】
・週1回「ICT活用Tips」を職員室に掲示
・月1回「今月のおすすめアプリ」を紹介
・季節やイベントに合わせた活用アイデア提供
・他校の実践事例を定期的に共有
```

**個別カスタマイズ情報**
```
【受動的】同じ情報を全員に配る
【能動的】
・各先生の担当学年・教科に合わせた情報提供
・その先生の授業スタイルに合ったツール提案
・過去の相談内容を踏まえたフォローアップ
```

#### D. トラブル予防での能動的支援

**日常点検**
```
・毎朝の機器動作確認
・週1回の充電状況チェック
・月1回の詳細点検
・問題の早期発見と対応
```

**予防的メンテナンス**
```
・定期的なソフトウェアアップデート
・不要ファイルの整理
・セキュリティ設定の確認
・消耗品の在庫管理と発注提案
```

**ドキュメント整備**
```
・よくあるトラブルのFAQ作成
・機器の取扱説明書を分かりやすく再編集
・緊急時の連絡先一覧
・トラブル対応フローチャート
```

## 4. なぜ自発的・能動的支援が重要なのか

### 4-1. 先生方にとってのメリット

#### メリット1: 安心感が生まれる
```
「困った時に助けてくれる人」
    ↓
「困る前に支えてくれる人」
    ↓
「一緒に授業を作ってくれるパートナー」
```

- トラブルへの不安が減る
- 新しいことに挑戦しやすくなる
- ICT活用が日常化する

#### メリット2: 時間が節約できる
```
【受動的支援の場合】
問題発生 → 支援員を探す → 説明 → 対応待ち → 解決
（20分）

【能動的支援の場合】
問題が起きない → すぐに授業開始
（0分）
```

#### メリット3: 授業の質が向上する
- 技術的な不安がなくなる
- ICTの効果的な活用方法が分かる
- 子どもたちの学びに集中できる

### 4-2. 子どもたちにとってのメリット

- 授業がスムーズに進む
- 待ち時間が減る
- より豊かな学習体験ができる
- デジタルスキルが向上する

### 4-3. 学校全体にとってのメリット

- ICT活用率が向上する
- トラブル対応時間が減る
- 教職員の負担が軽減される
- 学校全体のICTリテラシーが向上する
- 先進的な取り組みができる

### 4-4. ICT支援員自身にとってのメリット

#### メリット1: 信頼関係が構築される
```
「困った時だけ呼ばれる人」
    ↓
「いつも気にかけてくれる人」
    ↓
「学校になくてはならない存在」
```

#### メリット2: やりがいが増す
- 感謝される機会が増える
- 目に見える成果が出る
- 自分の存在価値を実感できる

#### メリット3: スキルアップにつながる
- 予測力が身につく
- 提案力が磨かれる
- 問題解決力が向上する
- コミュニケーション力が伸びる

#### メリット4: 仕事が効率化される
- 緊急対応が減る
- 計画的に動ける
- 同じトラブルが減る
- 余裕を持って業務に取り組める

## 5. 自発的・能動的支援を実践する方法

### 5-1. マインドセット（考え方）の転換

#### 転換1: 「依頼待ち」から「先読み」へ
```
【Before】
「今日は依頼がないから暇だな」

【After】
「今日は余裕があるから、来週の準備をしよう」
「各教室を回って、困っていることがないか聞いてみよう」
```

#### 転換2: 「個別対応」から「全体最適」へ
```
【Before】
A先生の問題を解決して終わり

【After】
A先生の問題 → 他の先生も同じ問題を抱えているかも
→ 全体に情報共有 → マニュアル作成 → 研修実施
```

#### 転換3: 「技術者」から「教育パートナー」へ
```
【Before】
「私は機械の専門家」

【After】
「私は、ICTを使って先生方の授業をサポートする
教育チームの一員」
```

### 5-2. 具体的な実践ステップ

#### ステップ1: 情報収集の習慣化

**毎日やること**
```
□ 朝礼・職員会議への参加
□ 週案・時間割の確認
□ 先生方との雑談（情報収集）
□ 校内巡回（機器の状態確認）
```

**週1回やること**
```
□ 翌週の行事・授業予定の確認
□ 機器の動作確認
□ ICT活用状況の振り返り
□ 必要な準備のリストアップ
```

**月1回やること**
```
□ 翌月の行事カレンダー確認
□ 消耗品の在庫チェック
□ 新しい情報・ツールの収集
□ 研修企画の立案
```

#### ステップ2: 予測力を磨く

**パターン認識**
```
「運動会の前には、ビデオカメラの準備依頼が増える」
「学期末には、成績処理のサポート依頼が増える」
「新学期は、アカウント関連の質問が増える」
```

→ パターンが分かれば、事前準備ができる

**先生方の特性理解**
```
A先生：新しいツールに積極的 → 最新情報を優先的に提供
B先生：慎重派 → 実績のある方法を丁寧に説明
C先生：多忙 → 簡単・短時間でできる方法を提案
```

**季節・時期の把握**
```
4月：新学期準備、環境設定
5-6月：授業が本格化、ICT活用支援
7月：学期末、データバックアップ
8月：夏休み、機器メンテナンス
9月：2学期準備、研修実施
10-11月：行事対応、公開授業支援
12月：学期末、年末整理
1月：3学期準備
2-3月：年度末、データ移行、次年度準備
```

#### ステップ3: 声かけの工夫

**タイミング**
```
✅ 良いタイミング
・授業の前後（先生が教室にいる時）
・空き時間（職員室で余裕がある時）
・昼休み（リラックスしている時）

❌ 避けるべきタイミング
・授業中（緊急時以外）
・会議中
・明らかに忙しそうな時
```

**声かけの例文**

【初対面・関係構築期】
```
「おはようございます。何かお手伝いできることはありませんか？」
「今週の授業予定、拝見しました。ICTを使う予定はありますか？」
「いつでもお声がけください。職員室にいますので」
```

【関係ができてきたら】
```
「来週の社会の授業、地図アプリを使うと便利かもしれません」
「他のクラスでこんな実践がありました。参考になりますか？」
「新しい機能が追加されたので、お時間ある時にご紹介しますね」
```

【さらに深い関係になったら】
```
「次の単元、こんな授業デザインはいかがでしょう？」
「先生のクラスなら、この方法が合いそうですね」
「前回の授業、こんな改善案を考えてみました」
```

#### ステップ4: 提案力を高める

**提案の3原則**
```
1. 具体的に
   ×「ICTを使いましょう」
   ○「この単元なら、Google Earthで実際の地形を
      見せると分かりやすいです」

2. メリットを明確に
   「この方法なら、○○の時間が△分短縮できます」
   「子どもたちの理解度が○○%向上した事例があります」

3. 選択肢を示す
   「AとBの方法があります。先生の授業スタイルなら
    Aが合うかもしれませんが、いかがでしょうか？」
```

**提案の段階的アプローチ**
```
【段階1】情報提供
「こんな方法があります」

【段階2】メリット提示
「この方法だと○○ができます」

【段階3】具体的提案
「次の授業で試してみませんか？準備します」

【段階4】一緒に実践
「一緒にやってみましょう」

【段階5】振り返りと改善
「次回はこう改善しましょう」
```

#### ステップ5: ドキュメント化

**作るべきドキュメント**
```
□ 授業支援記録シート
  （いつ、誰に、何を、どう支援したか）
□ よくある質問FAQ
□ 機器トラブル対応マニュアル
□ 活用事例集
□ 年間サポートカレンダー
□ 機器配置図・ネットワーク構成図
```

**ドキュメントの活用**
- 同じ質問への迅速な対応
- 知識の蓄積と共有
- 後任への引継ぎ
- 自分の成長記録

### 5-3. 実践的なテクニック

#### テクニック1: 「3分ルール」
```
先生と会った時、3分以内で価値を提供する

例：
「先生、おはようございます。
 来週の理科の授業、タブレットで実験動画を撮ると
 振り返りに使えますよ（30秒）
 設定は簡単で、このボタンだけです（1分）
 必要なら事前に練習しましょうか？（30秒）
 では、良い1日を！」

→ 短時間で有益、負担にならない
```

#### テクニック2: 「プラス1提案」
```
依頼された内容 + 関連する提案

例：
先生：「プロジェクターの使い方教えて」
支援員：「かしこまりました。（依頼対応）
        ところで、このアプリを使うと
        子どもたちの端末から画像を
        プロジェクターに直接送れますよ。
        （プラス1提案）
        授業で使えそうなら、また説明しますね」
```

#### テクニック3: 「見える化」
```
自分の支援を可視化する

・月次レポート作成
  「今月の支援実績：授業支援○回、トラブル対応○件、
   研修実施○回、参加者満足度○%」
  
・活用事例の掲示
  「今月のICT活用ベストプラクティス」
  写真付きで職員室に掲示

・Before/After
  「この改善で、準備時間が15分→5分に短縮」
```

#### テクニック4: 「小さな成功体験の積み重ね」
```
いきなり大きな提案はしない
小さな成功から始める

例：
【×ダメな例】
「来月から全クラスでGoogle Classroom導入しましょう！」
→ 先生方の負担感大、抵抗感

【○良い例】
1週目：1クラスで課題配信を試す
2週目：成果を共有、他のクラスにも展開
3週目：使いたい先生が増える
1ヶ月後：自然に全クラスで活用

→ 段階的、無理がない、自発的な広がり
```

#### テクニック5: 「失敗を学びに変える」
```
トラブルや失敗を予防策に転換

例：
トラブル発生
  ↓
原因分析
  ↓
予防策を考案
  ↓
マニュアル化・共有
  ↓
同じトラブルが起きなくなる

「失敗は改善のチャンス」
```

## 6. 段階別アプローチ（初心者→中級者→上級者）

### 6-1. 【初心者レベル】関係構築と基礎固め（1-3ヶ月）

#### 目標
- 先生方に顔と名前を覚えてもらう
- 基本的な機器操作をマスターする
- 学校の1日の流れを理解する

#### やるべきこと

**毎日**
```
□ 朝の挨拶回り（職員室・各フロア）
□ 昼休みに職員室で雑談
□ 困っている先生がいないか観察
```

## 8. よくある失敗と対策

### 失敗1: 「押し付け」になってしまう

**失敗例**
```
支援員「この方法が絶対良いです！やりましょう！」
先生「（うーん、今は忙しいんだけど…）」
  ↓
先生が避けるようになる
```

**原因**
- 相手の状況を考えていない
- 一方的な提案
- タイミングが悪い

**対策**
```
✅ 相手の状況を確認
  「今、お時間よろしいですか？」
  「お忙しいと思いますが…」

✅ 提案は選択肢形式で
  「○○という方法もありますが、いかがですか？」
  「無理にとは言いませんが…」

✅ 断られても引き下がる
  「分かりました。また何かあればお声がけください」
```

---

### 失敗2: 「やりすぎ」で疲弊する

**失敗例**
```
支援員「全部自分でやらなきゃ！」
  ↓
毎日残業、休日出勤
  ↓
燃え尽き症候群
```

**原因**
- 完璧主義
- 断れない性格
- 優先順位がつけられない

**対策**
```
✅ 「重要度×緊急度」で優先順位をつける

【最優先】
・授業に直結すること
・安全に関わること
・複数人に影響すること

【後回し】
・緊急性の低い依頼
・個人的な趣味レベルの質問
・自分で調べられること

✅ できないことは正直に伝える
  「申し訳ございませんが、この件は対応できません」
  「○○なら対応できます」

✅ 先生方にも「自走」してもらう
  「マニュアルがあるので、まず試してみてください」
  「それでもダメなら呼んでください」
```

---

### 失敗3: 「空回り」する

**失敗例**
```
支援員「この資料、役立つと思って作ったのに…」
  ↓
誰も読んでくれない、使われない
  ↓
「自己満足だったのか…」
```

**原因**
- ニーズを把握していない
- 難易度が合っていない
- 伝え方が悪い

**対策**
```
✅ 事前にニーズ確認
  「こんな資料があったら便利ですか？」
  「どんな情報が欲しいですか？」

✅ 小さく試す
  いきなり大量の資料を作らない
  まず1枚、試しに配ってみる
  反応を見て改善

✅ フィードバックをもらう
  「この資料、分かりやすかったですか？」
  「他に知りたいことはありますか？」
```

---

### 失敗4: 「遠慮しすぎ」で存在感が薄い

**失敗例**
```
支援員「自分なんかが提案していいのかな…」
  ↓
何も言わず、じっと待つ
  ↓
「あの人、何してるの？」と思われる
```

**原因**
- 自信がない
- 遠慮しすぎ
- 受け身の姿勢

**対策**
```
✅ 小さなことから始める
  「おはようございます」の挨拶から
  「何かお手伝いできることは？」の声かけ

✅ 実績を積む
  小さな支援でも確実に成果を出す
  →自信につながる

✅ 「謙虚だが卑屈にならない」
  ×「私なんかが…」
  ○「もし良ければ…」「お役に立てれば…」
```

---

### 失敗5: 「技術偏重」になる

**失敗例**
```
支援員「この機能はこうで、こうすれば…」
先生「（難しすぎて分からない…）」
  ↓
先生がICTを敬遠するようになる
```

**原因**
- 技術的な説明が多すぎる
- 相手のレベルに合わせていない
- 目的を伝えていない

**対策**
```
✅ 「何ができるか」から説明
  ×「このアプリはクラウドベースで…」
  ○「このアプリを使うと、子どもたちの意見が
     リアルタイムで見られます」

✅ 相手のレベルに合わせる
  初心者：シンプルに、1つずつ
  中級者：応用的な使い方
  上級者：最新機能の紹介

✅ 「教育効果」を強調
  技術<教育
  「これを使うと○○が学べます」
```

```
【能動的支援の3つの柱】

1. 予測する力
   - 先を見通す
   - パターンを見つける
   - リスクを察知する

2. 提案する力
   - 適切なタイミング
   - 相手に合わせた内容
   - 実現可能な方法

3. 継続する力
   - 一度で終わらない
   - PDCAを回す
   - 改善し続ける
```

### 10-2. 段階的な成長

焦らず、段階的に成長しましょう。

```
【成長のステップ】

ステップ1（1-3ヶ月）
関係構築と基礎固め
→ 信頼される

ステップ2（4-12ヶ月）
予測と提案
→ 頼られる

ステップ3（1年以上）
学校全体の最適化
→ なくてはならない存在になる
```

### 10-3. 最も大切なこと

技術や知識よりも大切なのは、

**「先生方と子どもたちの役に立ちたい」という気持ち**

その気持ちがあれば、
- 自然と行動が能動的になります
- 先生方との関係が深まります
- やりがいを感じられます

### 10-4. 今日から始める一歩

**今日、今すぐできること：**

```
□ 先生方に「何かお手伝いできることはありませんか？」
  と声をかける

□ 明日の授業予定を確認する

□ 教室を巡回し、機器の状態を確認する

□ 1つでいいので、情報提供する
```

**小さな一歩が、大きな変化につながります。**
