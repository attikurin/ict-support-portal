EdTechツール比較

Google Workspace・Microsoft 365・Appleの教育向けプラットフォーム徹底比較と選定ガイド

このページで学べること

GIGAスクール構想により、全国の学校でGoogle Workspace、Microsoft 365、Appleといった主要プラットフォームが導入されています。本ページでは、各プラットフォームの特徴、機能比較、選定のポイントをICT支援員の視点から解説します。

学習目標チェックリスト

  • 3大プラットフォームの特徴と違いを理解する
  • 学習管理システム（LMS）の機能を比較できる
  • 協働学習ツールの使い分けができる
  • デバイスとの親和性を理解する
  • コスト・ライセンス体系を把握する
  • 学校のニーズに合わせた選定ができる
  • 移行・切り替え時の注意点を理解する

1. 3大教育プラットフォーム概要

1.1 Google Workspace for Education

基本情報

  • 提供元: Google LLC
  • 対象デバイス: Chromebook、Windows、Mac、iPad、Android
  • 主要サービス: Classroom、Meet、Drive、Gmail、ドキュメント、スプレッドシート、スライド
  • LMS: Google Classroom
  • 無料版: Education Fundamentals（児童生徒用アカウント無制限）

Google Workspaceの特徴

  • シンプルで直感的: 学習曲線が緩やか、初心者でも使いやすい
  • リアルタイム共同編集: 複数人が同時に編集可能
  • Chromebookとの親和性: 起動が早く、軽量で低価格
  • クラウドネイティブ: すべてブラウザで完結
  • 豊富な拡張機能: Chrome拡張機能やアドオンが充実
  • 自動保存: データ損失のリスクが低い

1.2 Microsoft 365 Education

基本情報

  • 提供元: Microsoft Corporation
  • 対象デバイス: Windows、Mac、iPad、Chromebook、Android
  • 主要サービス: Teams、OneDrive、Outlook、Word、Excel、PowerPoint、OneNote
  • LMS: Microsoft Teams for Education
  • 無料版: Microsoft 365 A1（教職員・児童生徒用）

Microsoft 365の特徴

  • Office互換性: Word/Excel/PowerPointとの完全互換
  • 統合プラットフォーム: Teamsで授業・課題・コミュニケーションを一元管理
  • Windows端末との親和性: OSレベルでの統合
  • 高度な機能: 専門的な機能（マクロ、高度な数式等）
  • 企業利用との連続性: 社会に出ても使えるスキル
  • OneNote: デジタルノートとして高機能

1.3 Apple School Manager + iWork

基本情報

  • 提供元: Apple Inc.
  • 対象デバイス: iPad、Mac、iPhone
  • 主要サービス: Schoolwork、Classroom、Pages、Numbers、Keynote、iCloud
  • LMS: Apple Schoolwork
  • コスト: アプリは無料、デバイスは高価

Appleの特徴

  • 直感的なUI: デザイン性が高く、操作が直感的
  • クリエイティブ重視: 動画編集（iMovie）、音楽制作（GarageBand）に強い
  • iPadの教育機能: Apple Pencilによる手書き、AR学習
  • アクセシビリティ: 特別支援教育向け機能が充実
  • セキュリティ: プライバシー保護に強いこだわり
  • 統一されたエコシステム: Apple製品間のシームレスな連携

3. デバイスとの親和性

3.1 Chromebook環境

Chromebookに最適なプラットフォーム: Google Workspace

  • 完全統合: Chrome OSとGoogle Workspaceはシームレス
  • 高速起動: 5秒で起動、すぐに作業開始
  • 低スペックでも快適: クラウド処理のため端末負荷が少ない
  • 自動更新: OSとアプリが常に最新
  • 管理が容易: Google Admin Consoleで一元管理

Chromebookでの他プラットフォーム利用

  • Microsoft 365: ブラウザ版Officeは利用可能だが、一部機能に制限
  • Apple: Pages/Numbers/KeynoteはiCloud.com経由で利用可能（機能制限あり）

3.2 Windows環境

Windows端末に最適: Microsoft 365

  • OS統合: Windows 11とMicrosoft 365の深い統合
  • デスクトップアプリ: フル機能のOfficeアプリが利用可能
  • Active Directory連携: 校務システムとの統合が容易
  • 高性能: 高度な処理やグラフィック作業に対応

Windowsでの他プラットフォーム利用

  • Google Workspace: ブラウザで完全利用可能、問題なし
  • Apple: iCloud for Windowsで一部連携可能

3.3 iPad環境

iPadに最適: Apple + Google/Microsoft

  • Apple純正: Pages/Numbers/Keynote、Schoolworkが完全統合
  • Google/Microsoftアプリ: どちらも高品質なiPadアプリを提供
  • Apple Pencil: 手書き学習に最適
  • マルチタスク: Split Viewで複数アプリ同時利用

iPadでの推奨構成

  • パターン1（Apple中心）: Schoolwork + Pages/Numbers/Keynote
  • パターン2（Google中心）: Classroom + Google Driveアプリ
  • パターン3（Microsoft中心）: Teams + Office Mobileアプリ
  • ハイブリッド: Classroom（LMS） + Pages（文書作成）の組み合わせも可

4.2 Microsoft 365 Education

プラン・料金体系

プラン名 料金 主な機能
Microsoft 365 A1 無料 Teams、Office Online、OneDrive（1TB）
Microsoft 365 A3 約350円/ユーザー/月 デスクトップOffice、高度なセキュリティ
Microsoft 365 A5 約700円/ユーザー/月 A3機能 + 高度な分析、音声通話

4.3 Apple

料金体系

  • Apple School Manager: 無料（アカウント管理システム）
  • Schoolwork: 無料（LMS）
  • iWork（Pages/Numbers/Keynote）: 無料
  • iCloudストレージ:
    • 5GB: 無料
    • 50GB: 130円/月
    • 200GB: 400円/月
    • 2TB: 1,300円/月
  • デバイスコスト: iPad（約5万円〜）、Mac（約15万円〜）が必要

トータルコスト比較の視点

  • 初期費用: デバイス購入費（Chromebook: 3-5万円、Windows: 5-8万円、iPad: 5-10万円）
  • ランニングコスト: ライセンス料、ストレージ追加費用
  • 管理コスト: MDM（モバイルデバイス管理）、トラブル対応の人件費
  • 更新サイクル: デバイスの耐用年数（通常4-5年）
  • トレーニングコスト: 教職員研修、ICT支援員配置

5. プラットフォーム選定ガイド

5.1 学校の状況別おすすめプラットフォーム

Google Workspaceが向いている学校

  • 初めてのICT導入: シンプルで教職員・児童生徒が習得しやすい
  • 予算が限られている: 無料プランでも充実、Chromebookは低価格
  • 協働学習重視: リアルタイム共同編集が優秀
  • クラウド中心: インターネット環境が整備されている
  • 小学校: 直感的な操作で低学年でも使いやすい

Microsoft 365が向いている学校

  • 既存のWindows環境: 既存資産を活かせる
  • Office互換性重視: 社会に出ても使うスキル
  • 高度な機能が必要: 専門教科（商業・工業等）
  • 校務システムとの連携: Active Directory統合
  • 中学校・高等学校: より高度な学習に対応

Appleが向いている学校

  • クリエイティブ重視: 動画・音楽・アート教育
  • 特別支援教育: アクセシビリティ機能が充実
  • BYOD環境: 保護者所有のiPad/Mac活用
  • デザイン重視: 直感的なUI、美しいデザイン
  • 私立学校: ブランドイメージ、差別化

5.2 判断基準チェックリスト

プラットフォーム選定時の確認項目

  • 予算: 初期費用とランニングコストの合計は？
  • 既存環境: 現在のデバイス・システムとの親和性は？
  • 教職員のスキル: ITリテラシーのレベルは？
  • 学習目標: どのような授業・学習を実現したいか？
  • 学校規模: 児童生徒数、教職員数は？
  • ネットワーク環境: インターネット回線の速度・安定性は？
  • サポート体制: ICT支援員の配置、ベンダーサポートは？
  • 保護者の理解: 家庭での利用、BYOD対応は？
  • 将来性: 5年後も継続利用できるか？

6. プラットフォーム移行・切り替え

6.1 移行が必要になるケース

  • デバイス更新: GIGAスクール第2期でデバイス変更
  • 自治体方針変更: 教育委員会の統一方針
  • コスト削減: ライセンス費用の見直し
  • 機能不足: 現在のプラットフォームでは実現できない授業
  • 統合・合併: 学校統廃合による環境統一

6.2 移行時の注意点

データ移行

  • ファイル形式: Googleドキュメント ⇔ Word等の変換
  • クラス設定: LMS上のクラス・メンバー情報の移行
  • 課題履歴: 過去の課題・成績データのエクスポート
  • 移行ツール: Google Workspace Migration for Microsoft Exchange (GWMME) 等の活用

人的対応

  • 教職員研修: 新プラットフォームの使い方講習（最低3回）
  • 児童生徒への説明: 操作方法、アカウント切り替え
  • 保護者への周知: 変更理由、家庭での対応
  • 移行期間: 夏休み等の長期休暇を活用
  • 並行運用期間: 1-2ヶ月は旧環境も維持

6.3 移行スケジュール例

・教職員向けフォローアップ研修
・旧環境の段階的停止

7. ICT支援員の役割

7.1 選定プロセスでの支援

支援内容

  • 情報提供: 各プラットフォームの特徴・機能を客観的に説明
  • デモンストレーション: 実際の操作を見せて比較
  • 他校事例紹介: 近隣校・類似校の導入事例を共有
  • コスト試算: 5年間のトータルコスト計算
  • 教職員の意見集約: アンケート・ヒアリング実施

7.2 導入・運用での支援

継続的な支援

  • 初期設定: アカウント作成、クラス設定
  • マニュアル作成: 学校独自の使い方ガイド
  • 定期研修: 学期ごとのスキルアップ研修
  • 個別サポート: 教員ごとの困りごとに対応
  • 活用促進: 新機能の紹介、活用事例の横展開
  • トラブル対応: 技術的問題の迅速な解決
